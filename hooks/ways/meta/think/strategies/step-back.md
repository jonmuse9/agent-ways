# Step-Back

## Signature
keywords: stuck step-back general principle abstract first-principles why fundamentally bigger-picture zoom-out
## Stages

### 1. Problem Reception
State the specific problem you're stuck on. What have you tried? Where did it break down?

### 2. Abstraction
Step back: "What general principle or pattern governs this kind of problem?" Don't think about the specific code or system — think about the category of problem.

### 3. Principle Articulation
State the principle clearly. Give 1-2 examples from other domains where the same principle applies. If you can't articulate a clear principle, you haven't abstracted enough — go broader.

### 4. Specific Application
Apply the principle back to the original problem. The principle should suggest an approach that wasn't visible from the specifics alone.

### 5. Verification
Does the specific solution actually follow from the principle? Test it: if the principle is correct, what else should be true? Check those predictions.
